    Ft_putchar

    Pour ce premmier programme nous devons refactoriser la fonction putcha() de la libc.

    Mais comment savoir que fait putchar()? Il suffit de demander à votre terminal. man putchar. Voilà ce que va vous répondre votre terminal: 

    PUTS(3)                                       Linux Programmer's Manual                                       PUTS(3)

NAME
       fputc, fputs, putc, putchar, puts - output of characters and strings

SYNOPSIS
       #include <stdio.h>

       int fputc(int c, FILE *stream);

       int fputs(const char *s, FILE *stream);

       int putc(int c, FILE *stream);

       int putchar(int c);

       int puts(const char *s);

DESCRIPTION
       fputc() writes the character c, cast to an unsigned char, to stream.

       fputs() writes the string s to stream, without its terminating null byte ('\0').

       putc()  is equivalent to fputc() except that it may be implemented as a macro which evaluates stream more than
       once.

       putchar(c) is equivalent to putc(c, stdout).

       puts() writes the string s and a trailing newline to stdout.

       Calls to the functions described here can be mixed with each other and with calls to  other  output  functions
       from the stdio library for the same output stream.

       For nonlocking counterparts, see unlocked_stdio(3).

RETURN VALUE
       fputc(), putc() and putchar() return the character written as an unsigned char cast to an int or EOF on error.

       puts() and fputs() return a nonnegative number on success, or EOF on error.

ATTRIBUTES
       For an explanation of the terms used in this section, see attributes(7).

       ┌──────────────────────────┬───────────────┬─────────┐
       │Interface                 │ Attribute     │ Value   │
       ├──────────────────────────┼───────────────┼─────────┤
       │fputc(), fputs(), putc(), │ Thread safety │ MT-Safe │
       │putchar(), puts()         │               │         │
       └──────────────────────────┴───────────────┴─────────┘
CONFORMING TO
       POSIX.1-2001, POSIX.1-2008, C89, C99.

BUGS
       It is not advisable to mix calls to output functions from the stdio library with low-level calls  to  write(2)
       for  the file descriptor associated with the same output stream; the results will be undefined and very proba‐
       bly not what you want.

SEE ALSO
       write(2), ferror(3), fgets(3), fopen(3), fputwc(3), fputws(3),  fseek(3),  fwrite(3),  putwchar(3),  scanf(3),
       unlocked_stdio(3)

    Nous pouvons voir que putchar() et une fonction qui prend une variable du type int le transforme dans le type unsigned char et l'affiche, puis la fonction nous renvoie un chiffre positif en cas de réussite et un chiffre négatif en cas d'échec.

    Nous pouvons tester la fonction par nous-mêmes. il suffit de crée un fichier C. Pour cela vim nom_du_fichier.c, écrire: 

      #include <stdio.h>
  
  int	main(void)
  {
  	putchar(54);
  	return (0);
  }

    Il ne reste plus qu'a compiler ce fichier. Pour cela gcc nom_du_fichier.c (Si vous voulez en apprendre plus sur gcc: man gcc)

    Avec la commande ls (man ls) vous pouvez remarquer qu'un fichier a.out et apparus. C'est l'executable de notre magnifique programme. Il vous suffit de faire : ./a.out

    Notre programme nous affiche donc le caractère 5 mais pourquoi 5 et pas 53 c'est tout simple la fonction putchar() prend une variable du type INT donc un entier positif ou négatif et le transforme en usigned char c’est-à-dire un caractère de la table ASCII (man ascii) donc 5 dans notre cas.

    Pour realiser notre propre putchar nous allons la prototyper comme la vrai c'est a dire: 

    int		ft_printf(int c);	//Cela est le prototypage de notre fonction

int		ft_printf(int c)	//Cela est la déclaration de notre fonction
{
	return (0);			//Ici c'est ce que va faire la fonction.
}

    Pour réaliser notre fonction nous aurons besoin d'une autre fonction de la libc appeler write() (man write). Nous devrrons inclure la lib unistd.h pour utiliser write(). 

    #include <unistd.h>			//Inclusion de la lib unistd.h

int		ft_putchar(int c);	//Prototypage de notre fonction

int		ft_putchar(int c)	//Déclaration de notre fonction
{
	int		d;		//Declaration de notre varriable de retour.
	d = write(1, &c, 1);		//Appel de la fonction write pour écrire sur la sortie standard.
					//d prend la valeur du retour de la fonction de write().
	return(d);			//Retour de notre varriable
}

int		main(void)
{
	ft_putchar(53);			//Test de notre nouvelle fonction
	putchar(53);			//Fonction de la libc
	return (0);
}

    Nous pouvons maintenant tester notre programme. Il suffit de le compiler et l'exécuter gcc nom_du_fichier.c ./a.out notre programme affiche sur la sortie standard "55".

    Realiser la fonction ft_print_alphabet.

  • Écrire une fonction qui affiche l’alphabet en minuscule sur une seule ligne, dans l’ordre croissant, à partir de la lettre ’a’.
  • Elle devra être prototypée de la façon suivante : void ft_print_alphabet(void);
  • Fonctions Autorisées : write

    • Realiser la fonction ft_print_reverse_alphabet.

  • Écrire une fonction qui affiche l’alphabet en minuscule sur une seule ligne, dans l’ordre décroissant, à partir de la lettre ’z’.
  • Elle devra être prototypée de la façon suivante : void ft_print_reverse_alphabet(void);
  • Fonctions Autorisées : write

    • Realiser la fonction ft_is_negative.

  • Écrire une fonction qui affiche ’N’ ou ’P’ suivant le signe de l’entier passé en paramètre. Si n est négatif alors afficher ’N’. Si n est positif ou nul alors afficher ’P’.
  • Elle devra être prototypée de la façon suivante : void ft_is_negative(int n);
  • Fonctions Autorisées : write

    • Realiser la fonction ft_print_comb.

  • Écrire une fonction qui affiche, dans l’ordre croissant, toutes les différentes combinaisons de trois chiffres différents dans l’ordre croissant - oui, la répétition est volontaire.
  • Cela donne: 
    ./a.out | cat -e
012, 013, 014, 015, 016, 017, 018, 019, 023, ..., 789$
  • 987 n’est pas là car 789 est déjà présent
  • 999 n’est pas là car ce nombre ne comporte pas exclusivement des chiffres différents les uns des autres
  • Elle devra être prototypée de la façon suivante : void ft_print_comb(void);
  • Fonctions Autorisées : write

    • Realiser la fonction ft_print_comb2.

  • Écrire une fonction qui affiche toutes les différentes combinaisons de deux nombres entre 0 et 99, dans l’ordre croissant.
  • Cela donne: 
    $>./a.out | cat -e
00 01, 00 02, 00 03, 00 04, 00 05, ..., 00 99, 01 02, ..., 97 99, 98 99$
  • Elle devra être prototypée de la façon suivante : void ft_print_comb2(void);
  • Fonctions Autorisées : write

    • Realiser la fonction ft_putnbr.

  • Écrire une fonction qui affiche un nombre passé en paramètre. La fonction devra être capable d’afficher la totalité des valeurs possibles dans une variable de type int.
  • Elle devra être prototypée de la façon suivante : void ft_putnbr(int nb);
  • Fonctions Autorisées : write
  • par exemple : ft_putnbr(42) affiche "42".

    • Realiser la fonction ft_print_combn.

  • Écrire une fonction qui affiche toutes les différentes combinaisons de n chiffres dans l’ordre croissant.
  • n sera tel que : 0 < n < 10
  • Si n = 2, cela donne quelque chose comme ça : 
    $>./a.out | cat -e
01, 02, 03, ..., 09, 12, ..., 79, 89$>
  • Elle devra être prototypée de la façon suivante : void ft_print_combn(int n);
  • Fonctions Autorisées : write
